## 2. 포스팅 순서 및 간격 규칙

### 기본 원칙
- 하루 최대 1개 메인 포스트 (예외: 런칭일/긴급 공지)
- 포스트 간 최소 20시간 간격 유지
- 휴식일을 전략적으로 배치 (D-1, D+13)
- 리플라이/인용 RT는 메인 포스트와 별도 (하루 3-5개 허용)

### 시간대 전략

| 시간 (UTC) | 시간 (KST) | 대상 | 용도 |
|------------|------------|------|------|
| 14:00 | 23:00 | 북미 오후 + 유럽 저녁 | 주요 포스트 (기본 시간) |
| 16:00 | 01:00 (+1) | 북미 저녁 피크 | 임팩트 높은 포스트 (포스트 3, 6) |
| 06:00 | 15:00 | 아시아 오후 + 유럽 오전 | 후속 포스트/RT (D+7 이후) |

### 스레드 포스트 간격
- 연결 스레드(교육 콘텐츠 등) 작성 시 전체를 한 번에 게시
- 개별 포스트는 스레드로 연결하지 않음 (각각 독립 포스트)

## 3. 커뮤니티 대응 가이드 (FAQ 답변 템플릿)

### Q1: FYRST가 뭐야? pump.fun이랑 뭐가 달라?
**A:** FYRST is a token launchpad on Solana where deployers must stake SOL collateral into escrow. If they rug, buyers are automatically refunded. We also track deployer reputation across wallets. Same bonding curve experience, but with structural accountability. Details at fyrst.xyz

### Q2: 담보가 최소 1 SOL이면 너무 적은 거 아니야?
**A:** 1 SOL is the minimum. Deployers can stake more, and higher collateral earns a higher reputation badge (Bronze/Silver/Gold/Diamond). On pump.fun, the cost of rugging is effectively zero. FYRST makes it non-zero, and reputation scoring adds a persistent cost to bad behavior.

### Q3: 환불은 어떻게 작동해?
**A:** If a deployer removes liquidity or triggers a rug event within the 24-hour safety period, the escrowed SOL is automatically distributed to buyers pro-rata (proportional to each buyer's investment). No manual claims needed. The smart contract handles it.

### Q4: 평판 점수는 어떻게 매겨져?
**A:** Deployer Reputation Score (0-100, grades A through F) is calculated from: number of past launches, rug history, average token lifespan, and cross-wallet analysis of linked wallets. The score is permanent and cannot be reset by creating a new wallet.

### Q5: $FYRST 토큰은 왜 필요해?
**A:** $FYRST token holders get tiered access: priority buying windows, access to high-reputation deployer launches, fee discounts, and reputation boosts. Five tiers from Free to Whale. Holding $FYRST aligns your incentives with the protocol's safety mission.

### Q6: pump.fun이 같은 기능 추가하면?
**A:** pump.fun profits from volume regardless of rug outcomes. Adding deployer collateral would reduce their launch volume, which directly reduces their fee revenue. Their incentive structure works against accountability. FYRST was designed from the ground up with accountability as the core mechanism, not an afterthought.

### Q7: 컨트랙트 감사 받았어?
**A:** Our escrow contracts follow Anchor security best practices. We are conducting phased TVL expansion and internal audits. Full third-party audit details will be shared as the protocol matures. Smart contract code will be open-sourced.

### Q8: 어떤 체인이야?
**A:** Solana. Built with Anchor Framework. Compatible with all major Solana wallets (Phantom, Solflare, Backpack, etc.).

### 답변 작성 규칙
- 항상 사실에 기반하여 답변
- 가격 예측, 수익 보장 절대 불가
- "DYOR" 남발 금지 -- 구체적 정보를 제공할 것
- 이모지 사용 금지
- 경쟁 프로젝트 비방 금지 (사실 비교만 허용)
- 모르는 질문은 "확인 후 답변드리겠습니다"로 응대

## 4. 위기 대응 가이드

### 시나리오 1: FUD (근거 없는 부정적 소문)

**대응 단계:**
1. 30분 내 상황 파악 (소문의 출처, 확산 범위, 내용 정확성)
2. 사실 확인 후 공식 포스트 게시
3. 감정적 반응 금지 -- 데이터와 사실로만 대응

**템플릿:**
```
We are aware of claims regarding [specific claim].

Here are the facts: [factual rebuttal with on-chain data or documentation].

Our escrow contracts are verifiable on-chain at [address]. We encourage independent verification.
```

### 시나리오 2: 스마트 컨트랙트 취약점 의심/보고

**대응 단계:**
1. 즉시 기술팀 알림 (15분 내)
2. 취약점 확인 시: 영향 범위 파악 -> 필요시 일시 중단 -> 수정 -> 재개
3. 투명한 사후 보고 게시

**템플릿 (조사 중):**
```
We have received a report regarding a potential vulnerability in [component].

Our team is actively investigating. User funds in escrow are secured by [mechanism].

We will provide an update within [timeframe]. Updates will be posted here.
```

**템플릿 (해결 후):**
```
Investigation complete regarding the reported [component] issue.

Finding: [description of finding]
Impact: [affected scope, if any]
Resolution: [what was done]
Status: [current status]

Full post-mortem: [link]
```

### 시나리오 3: $FYRST 가격 급락 (30%+ 하락)

**대응 원칙:**
- 가격에 대해 직접 언급하지 않음
- 프로토콜 메트릭(TVL, 런칭 수, 환불 성공률)에 집중
- 패닉 셀링 조장 또는 바닥 매수 유도 모두 금지

**템플릿:**
```
Protocol metrics update:

- Total collateral staked: [X] SOL
- Active launches: [X]
- Successful refunds processed: [X]
- Deployer reputation scores tracked: [X]

FYRST continues to operate as designed. The system enforces accountability regardless of market conditions.
```

### 시나리오 4: 경쟁 프로젝트 공격적 마케팅

**대응 원칙:**
- 반격하지 않음
- 자사 메트릭과 기능으로만 응대
- "우리는 경쟁이 아닌 구조에 집중합니다" 톤 유지

**템플릿:**
```
We focus on building verifiable accountability infrastructure.

Our on-chain data is public. Deployer collateral, reputation scores, and refund records are all verifiable.

We welcome any project that makes token launches safer for buyers.
```

### 시나리오 5: 환불 메커니즘 실패 또는 지연

**대응 단계:**
1. 즉시 인지 및 영향 범위 파악 (10분 내)
2. 영향받은 사용자에게 직접 소통
3. 수동 환불 프로세스 가동 (자동 실패 시)
4. 원인 분석 및 수정 후 사후 보고

**템플릿:**
```
We identified a delay in refund processing for [X] transactions.

Affected users: [scope]
Cause: [brief technical explanation]
Resolution: [manual processing initiated / fix deployed]
Expected completion: [timeframe]

All affected buyers will receive their full pro-rata refund. No user funds are at risk.
```
