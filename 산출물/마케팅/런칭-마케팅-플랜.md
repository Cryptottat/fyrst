# FYRST 런칭 마케팅 플랜

> 전략: 데브넷 시연 → 커뮤니티 빌딩 → $FYRST 토큰 + 메인넷 동시 오픈

---

## 왜 데브넷 먼저인가

- 메인넷 스텔스 런칭 시 DAU 0 → "유저 없는 플랫폼" 첫인상 → 회복 불가
- pump.fun 유저들은 새 플랫폼에 **신뢰**가 필요 → 데브넷에서 먼저 증명
- 실제 작동 영상 = 이 시장에서 신뢰의 전부 (docs만 있는 프로젝트 99%)
- 커뮤니티가 생긴 상태에서 메인넷 오픈 → Day 1 유동성 확보
- 피드백 받아서 메인넷 전에 수정 가능

**Day 5-7: 데브넷 시연 영상 (핵심 콘텐츠)**

### 영상 4개 촬영 가이드

**영상 1: 토큰 런칭 시연 (30초)**
- 화면: fyrst.fun/launch 페이지
- 동작: 토큰 이름/심볼 입력 → 담보 티어 선택 → 이미지 업로드 → LAUNCH 클릭
- 강조: 에스크로 예치 확인, SPL 토큰 민팅 확인 (Solscan 링크)
- 자막: "Every launch requires deployer collateral."
- 마지막 프레임: "devnet live. mainnet soon." + fyrst.fun

**영상 2: 매수/매도 시연 (30초)**
- 화면: fyrst.fun/token/[mint] 페이지
- 동작: SOL 입력 → BUY → 토큰 잔고 표시 → 50% SELL → SOL 반환
- 강조: 본딩커브 차트 실시간 변동, 수수료 표시
- 자막: "Trade on the bonding curve. Fair pricing, transparent fees."

**영상 3: 러그 감지 → 자동 환불 (30초)**
- 화면: 배포자 프로필 → 러그 마킹 → 환불 실행
- 동작: mark_rugged 호출 → 에스크로 상태 변경 → process_refund 실행
- 강조: 구매자 SOL 반환 확인
- 자막: "Rugged? Get your SOL back. Automatic."

**영상 4: 평판 시스템 (15초)**
- 화면: 배포자 프로필 페이지
- 동작: 평판 점수, 등급 뱃지 (Bronze/Silver/Gold/Diamond), 런칭 이력
- 강조: 러그 카운트 빨간색 표시
- 자막: "Every wallet carries a reputation. Choose wisely."

### 영상 제작 팁
- 포맷: 화면 녹화 (1080p, 16:9)
- 자막: 영어, 굵은 흰색 + 검정 외곽선

- ❌ "100x guaranteed" 류의 수익 약속
- ❌ 에어드랍 이벤트 (봇만 몰림, 리얼 유저 0)
