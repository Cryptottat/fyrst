# FYRST 솔라나 밈코인 메타 대응 전략 (최종 개발 지침서 v6)

> **문서 목적:** AI 에이전트가 이 문서를 읽고 즉시 온체인 프로그램(Rust), 백엔드, 프론트엔드를 에러 없이 안전하게 수정할 수 있도록 구체적인 구현 로직과 수학적 공식을 정의합니다. 또한 **최적의 로컬/메인넷 포크 테스트 환경 구축 가이드**를 포함합니다.

---

## 1. 에스크로 담보금 및 마감시간(Deadline) 자율화

배포자(Dev)가 런칭 시 담보금과 **'졸업 마감시간(Deadline)'**을 스스로 선택하게 합니다.

### 🛠️ 온체인 스마트 컨트랙트 수정 지침 (`programs/fyrst/src/`)
*   **상태 구조체 변경 (`state.rs`의 `EscrowVault`):**
    *   기존 `created_at: i64` 외에 **`deadline_timestamp: i64`** 필드를 추가합니다.
*   **`create_escrow` 인스트럭션 (`escrow.rs`):**
    *   클라이언트로부터 `duration_seconds` (예: 1시간=3600)를 인자로 받아 `escrow.deadline_timestamp = Clock::get()?.unix_timestamp + duration_seconds;` 로 저장합니다.
*   **담보금 하한선 수정 (`constants.rs`):**
    *   `MIN_COLLATERAL`을 1 SOL에서 **0.1 SOL (100,000,000 lamports)** 로 하향 조정합니다.

---

## 2. 졸업(Graduation) 및 환불(Refund) 로직의 완벽한 분리

오직 **"졸업 여부"**와 **"마감시간 초과 여부"**만으로 담보금의 향방을 결정합니다.

### 🛠️ 온체인 로직 수정 지침
*   **`release_escrow` (배포자의 담보금 회수 - `escrow.rs`):**
    *   조건: **`require!(curve.graduated == true)`** (졸업 성공 시 즉시 회수 가능)
*   **`process_refund` (유저의 보증금 사냥 - `refund.rs`):**
    *   조건: **`require!(curve.graduated == false)`** AND **`require!(now >= escrow.deadline_timestamp)`** (마감시간 종료 및 졸업 실패 시 유저들이 N빵)

---

## 3. 프레셔 스코어 (Pressure Score) 및 전광판 UI (Marquee)

### 🧮 Pressure Score 계산식
*   **공식:** `(에스크로 SOL / max(남은 시간(초), 1)) * 100`

### 🚀 서버 부하를 고려한 아키텍처 (Agent Action - Critical)
1.  **백엔드:** 점수를 계산하지 않고 `escrowSol`과 `deadlineTimestamp`만 내려줍니다.
2.  **프론트엔드:** 브라우저에서 `setInterval`을 이용해 1초마다 남은 시간을 구하고 점수를 계산하여 화면을 재정렬(Re-sorting)합니다.
3.  **UI:** 상단에는 🔥마감 임박 전광판, 하단에는 💎탑 에스크로 전광판을 흐르게 합니다.

---

## 4. 수수료 클레임 점진적 잠금 해제 (Progressive Fee Unlock)

### 🛠️ 온체인 스마트 컨트랙트 수정 지침 (`state.rs`, `protocol.rs`)
*   **상태 구조체 변경 (`BondingCurve`):**
    *   `max_reserve_reached: u64`, `total_deployer_fees: u64`, `claimed_deployer_fees: u64` 필드 도입.
*   **`claim_fees` 로직:**
    *   `unlocked_fees = (total_deployer_fees * max_reserve_reached) / GRADUATION_THRESHOLD`
    *   `amount_to_claim = unlocked_fees - claimed_deployer_fees` (Zero Division 및 Overflow 주의: `u128` 캐스팅 필수)

## 5. 졸업(Graduation) 이후 프로세스 및 차트 스위칭

### 🛑 졸업 시 거래 중단 및 차트 처리
*   **온체인:** `graduated = true`가 되는 순간 FYRST 내부에서의 매수/매도는 `require!(!curve.graduated)`에 의해 완벽히 차단됩니다.
*   **프론트엔드 (UI/UX):**
    *   거래가 막히면 UI를 **"🚀 Raydium으로 유동성 이전 중... (Migrating to Raydium)"** 상태로 전환합니다.
    *   마이그레이션이 완료되면, 차트 소스를 내부 데이터가 아닌 **DexScreener API나 GeckoTerminal API**로 교체하여 Raydium 차트를 띄워주어야 "차트 끊김" 현상을 방지할 수 있습니다.

## 6. Helius 의존성 제거 및 로컬 테스트 환경 최적화

이전 기획에서는 Helius API를 통해 배포자의 과거 러그풀 이력이나 초기 번들링을 추적하려 했으나, **"졸업 보증금(Graduation Bounty)" 모델로 룰을 변경하면서 Helius API의 복잡한 추적 기능이 더 이상 필요 없어졌습니다.**

### 🗑️ Helius 의존성 제거 (Agent Action)
*   **러그풀 감지 로직 삭제:** `service/src/services/helius.ts`의 `detectRugPull` 등 복잡한 지갑 추적 로직을 완전히 삭제합니다.
*   **오직 "결과(졸업 여부)"만으로 승부:** 데브가 번들링을 하든 말든, 24시간 내에 졸업을 못하면 담보금이 몰수되므로 굳이 Helius로 과거를 캘 필요가 없습니다.

### 🚀 완벽한 로컬 테스트 환경 (Mainnet Fork)
Helius 의존성이 사라졌기 때문에, 이제 로컬 테스트 환경에서 **"런칭 -> 매수/매도 -> 에스크로 해제/환불 -> 레이디움 졸업"**까지의 **모든 풀사이클(Full-cycle)을 100% 완벽하게 테스트**할 수 있습니다. Mocking(가짜 데이터)을 쓸 필요가 전혀 없습니다.

**로컬 테스트 실행 방법:**
```bash
# 1. 메인넷 RPC URL을 사용하여 로컬 검증자(Validator)를 포크 모드로 실행
solana-test-validator --url https://api.mainnet-beta.solana.com --reset

# 2. 내 로컬 지갑에 10,000 SOL 에어드랍
solana airdrop 10000

# 3. 로컬 환경에 FYRST 프로그램 배포
anchor deploy --provider.cluster localnet
```

**결론:** Helius를 걷어냄으로써 시스템이 훨씬 가벼워졌고, 로컬 포크 환경에서 레이디움 연동뿐만 아니라 **FYRST의 모든 핵심 기능(본딩 커브, 수수료 클레임, 환불 등)을 실제 메인넷과 똑같은 조건에서 초고속으로 테스트**할 수 있게 되었습니다.
