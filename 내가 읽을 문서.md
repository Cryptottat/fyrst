## 수수료 구조

**유저 총 부담: 거래당 1%**

```
유저가 1 SOL 매수/매도 시:
├── 0.5% (0.005 SOL) → 배포자 (deployer, claimable)
└── 0.5% (0.005 SOL) → Treasury (수수료 지갑)
    └── 30% → $FYRST 바이백 (Jupiter swap)
```

- 배포자는 토큰 상세 페이지에서 `[ CLAIM FEES ]` 버튼으로 수수료 클레임
- 바이백은 1분마다 자동 판단, 신규 수수료의 30%가 0.01 SOL 이상이면 실행
- 바이백은 `BUYBACK_TOKEN_MINT` 환경변수에 CA 입력 시 활성화

## 본딩 커브

```
가격 = BASE_PRICE + SLOPE × 현재공급량
     = 0.0001 SOL + 0.00000001 × 공급량
```

- 매수: SOL 투입 → SPL 토큰 민팅 (적분 가격 계산)
- 매도: SPL 토큰 소각 → SOL 반환
- 졸업: 리저브가 85 SOL 도달 시 자동 graduated
- 런칭 + 초기매수: 1개 트랜잭션으로 배치 처리 (팬텀 1번 서명)

### pump.fun과 비교 (리서치 완료, 2026-03-05)
| | FYRST (현재) | pump.fun |
|--|-------------|----------|
| 커브 타입 | Linear (base + slope × S) | Constant Product (x × y = k) |
| 시작 가격 | 0.0001 SOL | ~0.000000028 SOL |
| 공급 방식 | Mint-on-demand (무제한) | 1B 고정 (800M 판매) |
| 졸업 기준 | 85 SOL reserve | ~85 SOL |
| 가격 비율 | ~3,571배 높음 | 기준 |

> **참고**: 커브 개편 검토 중. 변경 시 온체인 프로그램 재배포 필요.

## 최근 주요 변경 (2026-03-05)

### 본딩커브 개편 리서치
- pump.fun 스타일 본딩커브(constant product, virtual reserves)와 FYRST 본딩커브(linear) 비교 분석 완료
- pump.fun: `virtual_sol=30 SOL`, `virtual_token=1.073B`, 800M 실제 판매, 졸업 시 ~85 SOL
- FYRST: `base_price=0.0001 SOL`, `slope=0.00000001`, mint-on-demand 방식
- FYRST 가격이 pump.fun보다 ~3,571배 높아서 P-MCAP(price × 1B) 표시 시도 → 비현실적 값 → revert 완료
- 향후 커브 파라미터 변경 시 온체인 재배포 필요 (롤백 태그: `pre-bonding-curve-overhaul`)

### About FYRST 페이지 추가 (`/about`)
