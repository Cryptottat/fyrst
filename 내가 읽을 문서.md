## 지갑 & 주소 정리

### 온체인 지갑 역할 (4개)

```
┌─────────────────────────────────────────────────────────┐
│                  ProtocolConfig (PDA)                     │
│                                                           │
│  Authority ─── 프로토콜 설정 변경 권한                      │
│  Treasury  ─── $FYRST 바이백+번 전용 금고                   │
│  Ops Wallet ── 서비스 수익 (자유 출금 가능)                  │
└─────────────────────────────────────────────────────────┘
  + Cranker Wallet ── 졸업 자동 실행 봇 (별도)
```

| 지갑 | 역할 | SOL 충전 필요? | 설명 |
|------|------|:-:|------|
| **Authority** | 관리자 | X (가스비만) | 프로토콜 설정 변경 (treasury/ops 주소, 졸업 기준 등). `update_treasury`, `update_ops_wallet` 호출 가능. |
| **Treasury** | 바이백 금고 | X (받기만 함) | 프로토콜 수수료의 **60%**가 자동 입금. $FYRST 바이백+소각 전용. **건들지 마.** |
| **Ops Wallet** | 서비스 수익 | X (받기만 함) | 프로토콜 수수료의 **40%**가 자동 입금. 자유롭게 출금 가능. 크랭커 지갑과 같은 주소로 설정하면 크랭커 자체 충전됨. |
| **Cranker** | 졸업 봇 | **메인넷: ~86 SOL** | 졸업 시 WSOL 임시 프론트(~85 SOL) + 가스비. 프론트 SOL은 프로그램이 상환하므로 실 손실 = 가스비(~0.000005 SOL)만. 하지만 순간적으로 85+ SOL이 필요. |

> **Ops Wallet = Cranker 추천**: 크랭커 지갑을 Ops Wallet으로 설정하면 거래 수수료 40%가 크랭커 지갑에 쌓임 → 크랭커 SOL 잔액 자동 보충.

## 수수료 구조

**유저 총 부담: 거래당 1%**

```
유저가 1 SOL 매수/매도 시:
├── 0.5% (0.005 SOL) → 배포자 (deployer, claimable)
└── 0.5% (0.005 SOL) → 프로토콜
    ├── 60% (0.003 SOL) → Treasury (바이백+소각 전용)
    │   └── 30% → $FYRST 바이백 (Jupiter swap)
    └── 40% (0.002 SOL) → Ops Wallet (서비스 수익, 자유 출금)
```

- 배포자 수수료는 프로그레시브 언락: `unlocked = total_fees × max_reserve / GRADUATION_THRESHOLD`
- 배포자는 토큰 상세 페이지에서 `[ CLAIM FEES ]` 버튼으로 수수료 클레임
- 바이백은 1분마다 자동 판단, 신규 수수료의 30%가 0.01 SOL 이상이면 실행
- 바이백은 `BUYBACK_TOKEN_MINT` 환경변수에 CA 입력 시 활성화
- 60/40 비율은 온체인 상수 `OPS_SHARE_BPS = 4000` (40%)로 고정

## 에스크로 시스템

배포자가 토큰 런칭 시 SOL 담보를 걸어 신뢰를 보증하는 장치.

```
런칭 시: 배포자 → [create_escrow] → EscrowVault PDA (SOL 잠금)
                                     (금액 + 데드라인 설정)

3가지 결과:
┌─ 졸업 성공 → [release_escrow] → 배포자에게 100% 반환
│
├─ 데드라인 경과 (홀더 없음) → [expire_escrow] → 50% 배포자 + 50% 프로토콜
│                                                 (프로토콜 50% 중 60% treasury + 40% ops)
│
└─ 데드라인 경과 (홀더 있음) → 바이어가 [process_refund] → 토큰 소각 + SOL 프로라타 환불
                               (커브 reserve에서 비율 계산)
```

- **최소 담보**: 0.01 SOL (프론트 티어: Iron 0.01, Bronze 1, Silver 5, Gold 10, Diamond 25)
- **기간 설정**: 최소 1분(데브넷) / 최대 7일. 프론트 프리셋: 1/3/5/15/30분 + 1/2/6/12/24h
- **expire_escrow**: 데드라인 지나면 누구나 호출 가능 (permissionless)
- **process_refund**: 바이어 본인만 호출. 자기 토큰 소각 → 비율만큼 SOL 돌려받음

## 본딩 커브 (Constant Product AMM — pump.fun 스타일)

```
virtual_token × virtual_sol = k (상수)

매수: tokens_out = virtual_token - k / (virtual_sol + sol_in)
매도: sol_out = virtual_sol - k / (virtual_token + tokens_in)
MCAP: virtual_sol × total_supply / virtual_token
```

### 초기값 (TOKEN_DECIMALS = 6)
| 파라미터 | 값 | 의미 |
|----------|-----|------|
| virtual_token_reserves | 1,073,000,000,000,000 | 1.073B 토큰 (가격 계산용) |
| virtual_sol_reserves | 30,000,000,000 | 30 SOL |
| real_token_reserves | 793,100,000,000,000 | 793.1M 판매 가능 |
| token_total_supply | 1,000,000,000,000,000 | 1B 고정 공급 |

- 매수: SOL 투입 → 토큰 전송 (curve ATA → buyer ATA)
- 매도: 토큰 반환 (seller ATA → curve ATA) → SOL 지급
- 졸업: real_token == 0 (전량 판매) 또는 reserve >= 5 SOL (데브넷)
- 시작 MCAP: ~$3.6K, 졸업 MCAP: ~$69K (SOL $130 기준)
- 런칭 + 초기매수: 1개 트랜잭션으로 배치 처리 (팬텀 1번 서명)

> **변경 이력**: 2026-03-05 Linear curve → Constant Product AMM 전환 완료. 데브넷 재배포 필요.
